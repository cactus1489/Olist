# Olist 배송 지연 분석 대시보드 제작 작업지시서

본 문서는 Olist 이커머스 데이터 분석 결과를 바탕으로 하는 Streamlit 대시보드 제작을 위한 가이드라인입니다.

## 1. 프로젝트 개요
- **목적**: 브라질 Olist 이커머스 데이터를 활용하여 배송 지연 현황을 시각화하고, 지역/판매자/카테고리별 성과 분석 및 비교 기능을 제공하는 대시보드 구축.
- **주요 도구**: Python, Streamlit, Plotly (시각화 라이브러리).

## 2. 주요 기능 요구사항

### 2.1 키워드 기반 비교 기능
- **검색 및 비교**: 사용자가 검색창에 지역(State), 판매자 ID, 또는 상품 카테고리명을 입력하여 여러 대상을 멀티 셀렉트하고 비교할 수 있는 대시보드 상단 혹은 사이드바 인터페이스 구성.
- **동적 필터링**: 선택된 키워드에 따라 그래프와 표가 실시간으로 업데이트되어야 함.

### 2.2 메인 대시보드 구성 (기초 EDA 포함)
최소 **5개 이상의 그래프**와 **5개 이상의 표**를 포함하며, 다음의 탭 구조로 구성함.

#### [탭 1] 핵심 지표 및 현황 (Overview)
- **KPI 카드**: 총 거래건수, 총 거래금액, 총 구매자 수, 평균 지연율 등 요약 정보.
- **그래프 1**: 지역별 배송 지연 건수 (Map 또는 Bar 차트).
- **그래프 2**: 상품 카테고리별 거래 비중 (Pie 차트).
- **표 1**: 지역별 주요 지표 요약표.
- **표 2**: 배송 지연 발생 상위 TOP 10 판매자 리스트.

#### [탭 2] 배송 지연 상세 분석 (Delay Analysis)
- **그래프 3**: 주문 시점별 배송 지연 추세 (Line 차트).
- **그래프 4**: 예정일 대비 실제 배송일 차이 분포 (Histogram).
- **표 3**: 배송 상태별(delivered, canceled 등) 지연 건수 및 금액 상세표.
- **표 4**: 카테고리별 평균 배송 일수 및 지연율 순위표.

#### [탭 3] 판매자 및 상품 비교 (Comparison)
- **그래프 5**: 판매자별 매출액 및 지연 건수 상관관계 (Scatter 차트).
- **표 5**: 선택된 키워드(상품 또는 판매자) 간 직접 비교 분석 테이블.

### 2.3 데이터 용어 사전 탭 (Column Reference)
- **데이터 정리**: 데이터셋 내 각 컬럼의 고유값(Unique Values)을 중복 없이 정리하여 표출.
- **한글 번역**: 영문 컬럼명 및 주요 상태값(order_status 등)을 사용자가 이해하기 쉽게 **한글로 번역**하여 제공.
  - 예: `delivered` -> `배송 완료`, `customer_state` -> `고객 거주 주` 등.

## 3. 구현 세부 사항
- **디자인 가이딩**: Plotly의 다크 모드 또는 세련된 컬러 템플릿을 사용하여 시인성을 확보할 것.
- **성능 최적화**: 데이터 로딩 시 `@st.cache_data`를 활용하여 실행 속도를 높일 것.
- **데이터 소스**: `naver_api_mini_project/data/` 경로에 저장된 분석 결과 CSV 파일을 활용할 것.
